### 核心知识点回顾

#### 1. C++程序基础结构
```cpp
#include <iostream>  // 包含输入输出流库
using namespace std; // 使用std命名空间

int main() {          // 主函数入口
    cout << "Hello, world" << endl;
    return 0;         // 返回0表示正常退出
}
```
- **关键理解**：`main()` 是程序执行的起点；`cout` 用于标准输出；`endl` 输出换行并刷新缓冲区

#### 2. 常量定义方式
- **`#define`**：预处理指令，无类型检查，全局替换
- **`const`**：编译期常量，有类型检查，作用域可控，推荐优先使用

#### 3. 整数类型体系（重点掌握）

| 类型 | 大小(字节) | 有符号范围 | 无符号范围 | 字面量后缀 |
|------|-----------|-----------|-----------|-----------|
| `char` | 1 | -128 ~ 127 | 0 ~ 255 | - |
| `short` | 2 | -32768 ~ 32767 | 0 ~ 65535 | - |
| `int` | 4 | -2^31 ~ 2^31-1 | 0 ~ 2^32-1 | - |
| `long` | 4(Win) / 8(Linux64) | 平台相关 | 平台相关 | `l`/`L` |
| `long long` | 8 | -2^63 ~ 2^63-1 | 0 ~ 2^64-1 | `ll`/`LL` |

**补码截断机制**（核心难点）：
- 有符号：对 `2^n` 取模后，若结果 > `2^(n-1)-1`，再减去 `2^n`
- 无符号：直接对 `2^n` 取模
- 示例：`signed short num = 32768` → 截断后为 `-32768`

#### 4. 浮点类型
- **float**：4字节，6-7位有效数字，字面量后缀 `f`/`F`
- **double**：8字节，15-17位有效数字，默认浮点字面量类型
- **注意**：`float a = 3.14` 会触发隐式转换，建议写 `float a = 3.14f`

### 学习建议

#### 代码风格优化
1. **`using namespace std;` 的使用**：
   - 当前代码使用了 `using namespace std;`，在小型练习中没问题
   - **建议**：后续大型项目中避免使用，改用 `std::cout`、`std::endl` 等，防止命名冲突

2. **文件头部注释**：
   - **建议**：为每个练习文件添加文件头部注释，包含创建日期、学习主题、核心知识点

3. **整型固定宽度类型**：
   - **建议**：后续学习 `<cstdint>` 头文件，使用 `int8_t`、`int16_t`、`int32_t`、`int64_t` 等固定宽度类型，避免 `long` 的平台依赖性

4. **浮点精度控制**：
   - **建议**：学习 `<iomanip>` 头文件中的 `setprecision()` 函数，控制浮点数输出精度

#### 练习建议
1. **主动验证**：修改代码中的数值，观察输出结果，加深对补码截断的理解
2. **交叉平台测试**：如果条件允许，在不同操作系统上运行代码，观察 `long` 类型的大小差异
3. **边界值测试**：尝试各类型的最小值、最大值、溢出边界，验证自己的理解
